# Learning Workflow

After significant implementation, bug fixes, reviews, or incidents:

1. Identify what was learned.
2. Separate durable lessons from task-local details.
3. Update `KnowledgeBase.md` for reusable knowledge.
4. Update `DecisionLog.md` when a durable trade-off was chosen.
5. Update `Context.md` or `BusinessContext.md` when project understanding changed.
